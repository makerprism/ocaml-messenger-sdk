# CODE_REVIEW.md

### [CR-001] No secrets or sensitive data in code
- required: true
- applies_when: always
- pass_criteria: No API tokens, bot secrets, phone numbers, chat IDs, webhook secrets, or other credentials appear in the diff.
- evidence_required: Confirm no token-like strings, hardcoded secrets, or sensitive user data in changed files.

### [CR-002] Build must not be broken
- required: true
- applies_when: always
- pass_criteria: Changed `.ml`, `.mli`, `dune`, or `dune-project` files must be syntactically valid and not introduce obvious build errors.
- evidence_required: Review changed OCaml and dune files for syntax correctness.

### [CR-003] Opam files stay generated
- required: true
- applies_when: when `.opam` files change
- pass_criteria: Opam files must remain generated by dune. Manual edits to `.opam` files are not allowed; changes must go through `dune-project`.
- evidence_required: Confirm `.opam` file changes are consistent with `dune-project` package stanzas.

### [CR-004] No dune lockfile manual edits
- required: true
- applies_when: when `dune.lock/` files change
- pass_criteria: Lock files under `dune.lock/` must not be manually edited. They must be regenerated via `dune pkg lock`.
- evidence_required: Confirm lockfile changes are machine-generated, not hand-edited.

### [CR-005] Package isolation preserved
- required: true
- applies_when: when package dependencies or dune files change
- pass_criteria: Each package (messenger-whatsapp-cloud-v1, messenger-telegram-bot-v1, messenger-signal-bridge-v1) must declare its own dependencies and not rely on transitive dependencies from sibling packages.
- evidence_required: Review dependency declarations in `dune-project` and package `dune` files.
